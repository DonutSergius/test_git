Git command:
git revert - команда скасовує зміни, внесені певним комітом, і створює новий коміт зі скасованими змінами.
git reset - команда використовується для переміщення вказівника поточної гілки на інший коміт
git branch -  команда використовується для перегляду списку гілок, створення або видалення гілок
git checkout - команда використовується для переключення між гілками або відновлення файлів з певного коміту
git rebase - команда використовується для інтеграції змін з однієї гілки в іншу шляхом повторного застосування комітів на основі іншого базового коміту
git cherry-pick - команда використовується для застосування певних комітів з однієї гілки в іншу гілку
git commint --amend - команда використовується для зміни коміту
git reset --hard HEAD~1 - команда використовується для видалення останнього коміту
git bisect - команда використовується для знаходження багів в попередніх комітах, хороші коміти позначаються за допомогою git bisect good, погані за дпоомгою git bisect bad. Для того щоб повернутись до того коміту на якому користувач був до початку знаходження багу потрібно використати команду: git bisect reset
git submodule add - команда використовується для додавання іншого репозиторія як додаткового модуля до поточного проєкту. Ця команда створить посилання до іншого репозиторія в головному репозиторії проєкту
